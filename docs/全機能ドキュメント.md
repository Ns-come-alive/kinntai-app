# タイムカード（勤怠アプリ）｜全機能ドキュメント

夜間営業の店舗向け勤怠管理アプリです。キャストは名前をタップするだけで打刻でき、
管理者はシフト管理・遅刻/欠勤の自動判定・集計・スプレッドシート連携まで行えます。

- 技術スタック: Python (Flask) / PostgreSQL (Supabase) / HTML・CSS・JavaScript
- ホスティング: Render
- 外部連携: Google Apps Script（スプレッドシート連携）、Google Gemini（シフト画像読取）

---

## 1. 共通の仕組み

### 1.1 サイトアクセスゲート
- 全ページの前段に**アクセスコード**によるゲートがあります（`/gate`）。
- 正しいコードを入力すると Cookie が保存され、その端末では**実質無期限**で再入力不要（約20年）。
- 店舗のiPadや個人端末など、限られた端末だけに利用を絞る目的。

### 1.2 営業日（20:00〜翌9:00）
- **20:00〜翌9:00 を1営業日**として扱います。
  - 20:00以降 → その日の日付が営業日
  - 00:00〜08:59 → **前日**の日付が営業日
- 深夜をまたぐ勤務も同じ1日として正しく集計されます。

### 1.3 タイムゾーン
- すべての時刻処理は**日本時間（JST, UTC+9）**で統一。

### 1.4 コールドスタート対策
- `/healthz` エンドポイントを外部監視（cron-job.org）から定期アクセスし、
  無料プランのスリープによる「起動待ち画面」を抑制。

## 2. キャスト向け機能

### 2.1 タップログイン
- ログイン画面にキャスト名がボタン（タブ）で並び、**タップするだけでログイン**。
- キャスト側はパスワード不要。
- 上部に**日付・時刻をリアルタイム表示**。

### 2.2 打刻（出勤／同伴出勤／退勤）
- 「出勤」「同伴出勤」「退勤」ボタンで打刻。
- **退勤後の再出勤も可能**（出勤・退勤は何度でも押せる）。
- 打刻時刻は JST で記録。

### 2.3 遅刻判定と遅刻理由
- 管理者が登録したシフト開始時刻を過ぎて出勤すると**遅刻**。
- 遅刻時は**遅刻理由**を選択（交通機関の遅延／体調不良／その他）。
- **同伴出勤はシフト開始から1時間以内なら通常出勤**、超過で遅刻。
- **遅刻時間は30分刻み**で算出（例：21:00シフトに21:01出勤=0.5時間）。

### 2.4 個人の出勤履歴
- 月次集計（出勤日数・稼働時間・遅刻時間・欠勤日数）と日別明細を表示。

### 2.5 勤務時間の計算（15分刻み・切り捨て）
- 勤務時間（稼働時間）は**15分刻み（切り捨て）**で計算します。
- 15分に満たない端数は切り捨てます。
- 月間の「総稼働時間」は、各勤務を15分刻みにしたものを合算します。
- 出勤履歴・キャスト別集計・スプレッドシートのすべてでこのルールが適用されます。

| 実勤務時間 | 計算結果 |
|---|---|
| 1時間09分 | 1.0時間 |
| 1時間15分 | 1.25時間 |
| 1時間20分 | 1.25時間 |
| 1時間30分 | 1.5時間 |
| 1時間44分 | 1.5時間 |
| 1時間45分 | 1.75時間 |

> 参考：遅刻時間は**30分刻み（切り上げ）**で計算します（例：21:00シフトに21:01出勤＝0.5時間）。

## 3. 管理者向け機能

管理者は専用パスワードでログイン（`/login/admin`）。

### 3.1 管理ダッシュボード（`/admin`）
- その営業日の全キャストの打刻状況を一覧表示。

### 3.2 シフト管理（`/admin/shifts`）
- 営業日を選ぶと**自動でその日のシフトを表示**（表示ボタン不要）。
- 「◀ ▶」で前日・翌日へ移動。
- 出勤時刻は**プリセットボタン（21:00／22:00／23:00）**と「**休み**」で素早く登録。

### 3.3 シフト画像の自動読み取り（Gemini）
- シフト表の**画像をアップロード**すると、AI（Google Gemini）が内容を解析。
- `/admin/shifts/import` → プレビュー（`/admin/shifts/import/preview`）で確認・修正 → 保存（`/admin/shifts/import/save`）。
- 対応モデルは自動検出（`ListModels`）し、利用可能な最新の flash 系を選択。

### 3.4 欠勤の自動判定（`/admin/check-absent`）
- シフト登録があるのに**その営業時間内に出勤打刻がない**キャストを**欠勤（当欠）**として記録。

### 3.5 キャスト別の勤怠履歴（`/admin/history`）
- キャストのタブを選ぶと、**出勤日数・総稼働時間・遅刻時間・欠勤日数**と日別明細を表示。
- 対象月を指定可能。

### 3.6 キャスト管理（`/admin/casts`）
- キャストの**追加・削除**が可能。
- 変更は**ログイン画面のボタンに即時反映**（削除した人は消え、追加した人が現れる）。

### 3.7 CSVエクスポート（`/admin/export`）
- 勤怠データを CSV でダウンロード。

### 3.8 スプレッドシート書き出し（`/admin/sheets/export`）
- 指定した月のデータを**Googleスプレッドシートへ手動でまとめて書き出し**。

## 4. Googleスプレッドシート連携

### 4.1 仕組み
- 無料Googleアカウントでも動くよう、サービスアカウントではなく
  **Google Apps Script の Web アプリ（Webhook）**へ送信する方式（`sheets.py` → `apps_script.gs`）。
- 打刻・遅刻理由・退勤・欠勤判定などのたびに、当月の集計データを自動送信。
- 環境変数
  - `SHEETS_WEBHOOK_URL`：Apps Script の Web アプリURL（/exec）
  - `SHEETS_WEBHOOK_SECRET`：任意の合言葉（設定時のみ照合）

### 4.2 出力内容
- **全員分の月間集計タブ**（例：「月間集計 2026年07月」）のみ
  - キャスト・出勤日数・総稼働時間・遅刻時間・欠勤日数を一覧化
- ※ キャストごとの個別タブは作成しません。

### 4.3 見た目（`apps_script.gs`）
- 見出しは色帯、表はヘッダー色分け＋枠線。
- **列幅は内容の文字数（全角=2・半角=1）に応じて自動調整**。
  - 幅計算は `getDisplayValues()`（画面表示どおりの文字列）で行う。
    ※ `getValues()` だと日付/時刻セルが Date オブジェクト化して長い文字列となり、
    　営業日・出勤・退勤の列だけ過剰に広がる不具合があったため修正済み。

### 4.4 デプロイ確認
- ブラウザで `/exec` を開くと `kintai sheets webhook <バージョン>` が表示され、
  現在デプロイ中のコードのバージョンを確認できる。

## 6. データ項目（打刻履歴）

| 項目 | 説明 |
|---|---|
| 営業日 | 20:00〜翌9:00で区切った営業日 |
| 出勤 | 出勤打刻時刻（JST） |
| 退勤 | 退勤打刻時刻（JST） |
| 稼働(h) | 出勤〜退勤の稼働時間（15分刻み・切り捨て） |
| 種別 | 通常／同伴／当欠 など |
| ステータス | 通常／遅刻／同伴・遅刻／シフト未登録 など |
| 遅刻理由 | 交通機関の遅延／体調不良／その他 |

## 7. 環境変数まとめ

| 変数 | 用途 |
|---|---|
| `DATABASE_URL` | PostgreSQL（Supabase）接続文字列 |
| `SECRET_KEY` | Flask セッション用 |
| `SITE_ACCESS_COOKIE_MAX_AGE` | アクセスコード Cookie の保持秒数（既定：約20年） |
| `GEMINI_API_KEY` | シフト画像読み取り用 |
| `SHEETS_WEBHOOK_URL` | スプレッドシート連携（Apps Script /exec URL） |
| `SHEETS_WEBHOOK_SECRET` | スプレッドシート連携の合言葉（任意） |
